# BasicGO: простой стиль для учебных задач

При правках и новых примерах в этом репозитории (особенно `2.*`, строки, Stepik-задачи):

1. **Сначала самый простой вариант**, понятный новичку: `fmt.Scan` / один-два цикла / `strings.Count`, `strings.Index`, `strings.HasPrefix` и т.п., если условие задачи это допускает.
2. **Не усложнять по умолчанию:** не подключать `bufio`, `map`, `strings.Builder`, отдельные хелперы и «одна строка в main», пока нет явной нужды (пробелы во вводе, длинные строки, общий код для тестов).
3. **Ввод:** если в условии одна строка **без пробелов** — нормально `fmt.Scan(&s)`; если нужна **строка целиком с пробелами** — тогда `bufio` + строка до `\n`.
4. **Тесты:** те же идеи, что в решении; короткие `Test…` с очевидными кейсами; не раздувать абстракциями. Если функция нужна только ради тестов — одна маленькая функция с той же логикой, что в `main`, без «архитектуры».
5. **Пояснения:** коротко и по шагам; продвинутые варианты — только как опция с одной фразой «зачем» (скорость, полный ввод, проверка ошибок).
